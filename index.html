Research-Oriented University Information System

HTML Documentation for Main Project Classes

Project Overview

This project is a console-based Java application for a research-oriented university. It models users, courses, marks, lessons, research papers, research projects, authentication, requests, logs, and data storage. The project demonstrates core Object-Oriented Programming principles such as encapsulation, inheritance, polymorphism, abstraction, interfaces, enums, custom exceptions, collections, serialization, and design patterns.

Main packages: oopproject.users, oopproject.academic, oopproject.research, oopproject.core, oopproject.enums, and oopproject.exceptions.

User

Abstract classSerializable

User is the base abstract class for all user types in the system. It stores common information such as id, username, password, email, role, active status, inbox news, and direct messages.

Main Fields

  • id - unique user identifier.
  • username - login name.
  • password - user password.
  • email - user email address.
  • role - user role from UserType.
  • active - shows whether the user can log in.

Main Methods

  • checkPassword() - checks password and active status.
  • sendMessage() - sends a message through a mediator.
  • receiveMessage() - receives a direct message.
  • block() and unblock() - change user active status.
  • getId(), getUsername(), getEmail() - getter methods.
public abstract class User implements Serializable {
    protected int id;
    protected String username;
    protected String password;
    protected String email;
    protected UserType role;
    protected boolean active = true;
}

Student

Inherits UserCourse registration

Student extends User and represents a university student. It stores GPA, year, credits, failed courses, schedule, enrollments, transcript, teacher ratings, and research supervisor.

Main Fields

  • MAX_CREDITS - maximum allowed number of credits, equal to 21.
  • gpa - student GPA.
  • year - year of study.
  • credits - registered credits.
  • enrollments - list of registered course enrollments.
  • transcript - academic transcript.

Main Methods

  • registerCourse() - registers a student for a course.
  • viewTranscript() - returns student enrollments.
  • viewMarks() - returns available marks.
  • assignResearchSupervisor() - assigns a research supervisor.
  • rateTeacher() - stores teacher rating.
public void registerCourse(Course course)
        throws CreditLimitExceededException, AlreadyRegisteredException {
    if (credits + course.getCredits() > MAX_CREDITS) {
        throw new CreditLimitExceededException(
            "Student cannot register for more than " + MAX_CREDITS + " credits."
        );
    }
}

Employee

Inherits UserParent for Teacher and Manager

Employee extends User and represents a university employee. It stores salary, hire date, department, and employee requests. It is used as a parent class for Teacher and Manager.

Main Fields

  • salary - employee salary.
  • hireDate - date of hiring.
  • department - employee department.
  • requests - list of employee requests.

Main Methods

  • addRequest() - adds a request.
  • changeActive() - blocks or unblocks employee.
  • getDepartment() and setDepartment() - department access methods.

Teacher

Inherits EmployeeImplements Researcher

Teacher extends Employee and implements Researcher. A teacher can manage courses, view students, put marks, upload materials, create tasks, and participate in research activity. Professors are automatically researchers.

Main Fields

  • title - teacher title from TeacherType.
  • researcher - shows whether the teacher is a researcher.
  • researchSchool - research school name.
  • courses - assigned courses.
  • researchPapers - teacher's research papers.

Main Methods

  • putMark() - puts marks for students.
  • viewStudents() - returns students of a course.
  • uploadMaterial() - adds material to a course.
  • becomeResearcher() - changes teacher to researcher status.
  • addResearchPaper() - adds a paper and adds the teacher as author.
public class Teacher extends Employee implements Researcher {
    private TeacherType title;
    private boolean researcher;
    private final List<Course> courses = new ArrayList<>();
    private final List<ResearchPaper> researchPapers = new ArrayList<>();
}

Manager

Inherits EmployeeRequest management

Manager extends Employee. It manages course registration, assigns teachers, creates news, generates reports, and reviews requests.

Main Fields

  • type - manager type from ManagerType.
  • news - news created or managed by the manager.

Main Methods

  • assignTeacher() - assigns a teacher to a course.
  • openCourseForRegistration() - opens registration.
  • approveRequest() - approves a request.
  • rejectRequest() - rejects a request.
  • commentRequest() - adds a response comment.
public boolean approveRequest(Request request) {
    if (!isManagedRequest(request)) {
        return false;
    }
    request.approve();
    University.getInstance().addLog(this, "REQUEST_APPROVED id=" + request.getRequestId());
    return true;
}

Course

Academic modelSerializable

Course represents an academic course. It contains a name, code, number of credits, open status, instructors, lessons, materials, and enrollments. A course can have more than one instructor.

Main Fields

  • name - course name.
  • code - unique course code.
  • credits - course credit value.
  • open - registration status.
  • instructors - list of course teachers.
  • enrollments - student enrollments.

Main Methods

  • addStudent() - adds student enrollment.
  • addInstructor() - adds a teacher to the course.
  • findEnrollment() - finds student enrollment.
  • setOpen() - opens or closes registration.
public void addInstructor(Teacher teacher) {
    if (teacher != null && !instructors.contains(teacher)) {
        instructors.add(teacher);
    }
}

Researcher

InterfaceResearch abstraction

Researcher is an interface that describes research behavior. This design is flexible because a researcher can be a teacher, student, or another employee type. The system only requires the object to provide researcher behavior.

Main Methods

  • addResearchPaper() - adds a research paper.
  • removeResearchPaper() - removes a research paper.
  • getResearchPapers() - returns papers.
  • getResearcherName() - returns researcher name.
  • calculateHIndex() - calculates h-index.
  • printPapers() - prints sorted papers.
public interface Researcher extends Serializable {
    boolean addResearchPaper(ResearchPaper paper);
    boolean removeResearchPaper(ResearchPaper paper);
    List<ResearchPaper> getResearchPapers();
    String getResearcherName();
}

ResearchPaper

ComparableSerializable

ResearchPaper represents a scientific paper. It stores important information such as title, journal, publisher, pages, citations, publication date, DOI, keywords, and authors.

Main Fields

  • title - paper title.
  • journal - journal name.
  • publisher - publisher name.
  • pages - article length.
  • citations - citation count.
  • publishDate - publication date.
  • doi - digital object identifier.
  • authors - list of researchers.

Main Methods

  • addAuthor() - adds an author.
  • addCitation() - increases citation count.
  • compareTo() - compares papers by citations.
  • toDisplayString() - returns formatted paper information.
@Override
public int compareTo(ResearchPaper other) {
    return Integer.compare(other.citations, citations);
}

ResearchProject

Research modelCustom exception

ResearchProject represents a research project. It contains a topic, participants, and published papers. Only researchers can join a project. If a non-researcher tries to join, NonResearcherException is thrown.

Main Fields

  • topic - project topic.
  • participants - set of researchers.
  • papers - set of research papers.

Main Methods

  • addParticipant() - adds only researcher participants.
  • deleteParticipant() - removes participant.
  • addPaper() - adds a paper to the project.
  • getParticipants() - returns project participants.
public boolean addParticipant(Object candidate) throws NonResearcherException {
    if (!(candidate instanceof Researcher researcher)) {
        throw new NonResearcherException("Only researchers can join a research project.");
    }
    return addParticipant(researcher);
}

SingletonMessageMediatorSerializable

University is the central class of the system. It stores the main data collections and provides methods for user management, course management, research management, requests, messages, and logs. It uses the Singleton pattern so that the system has only one university instance.

Main Fields

  • courses - all courses.
  • researchers - all registered researchers.

Main Methods

  • addCourse() - adds a course.
  • registerStudentToCourse() - registers student for course.
  • putMark() - puts marks through teacher validation.
  • sendMessage() - sends direct messages.
  • addLog() - records system actions.
private static final University INSTANCE = new University();

OOP Principles in the Project

Encapsulation

Fields are hidden using private or protected, and access is provided through methods.

Inheritance

Student extends User, Employee extends User, Teacher extends Employee, and Manager extends Employee.

Polymorphism

The system works with User references and opens different menus depending on the actual runtime type of the user.

Abstraction

Researcher is an interface that abstracts research behavior.

Custom Exceptions

  • CreditLimitExceededException - prevents students from exceeding 21 credits.
  • LowHIndexException - prevents assigning a weak supervisor to a fourth-year student.
  • NonResearcherException - prevents non-researchers from joining research projects.
  • AlreadyRegisteredException - prevents duplicate course registration.

Design Patterns

  • Singleton: University has one shared instance.
  • Factory: UserFactory creates users based on UserType.
  • Mediator: MessageMediator controls message sending between users.
  • Data Store: DataStore saves and loads system state using serialization.

Enumerations

The project uses enums to represent fixed sets of values and make the code safer.

  • UserType - user roles.
  • TeacherType - teacher titles.
  • ManagerType - manager categories.
  • LessonType - lecture or practice.
  • ResearchSortType - date, citations, or length sorting.
  • RequestStatus and RequestType - request workflow values.
  • RoomType and DaysOfWeek - schedule-related values.
